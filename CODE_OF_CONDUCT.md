# Code of Conduct

## Expectations

The project is small, technical, and direct. Keep interactions the same way:

* be respectful and assume good intent
* critique code and behavior, not people
* keep discussion concrete, technical, and relevant to the project
* help newcomers understand context instead of gatekeeping it

## Unacceptable Behavior

The following are not welcome in issues, pull requests, discussions, or any project space:

* harassment, threats, or personal attacks
* discriminatory or demeaning language
* doxxing or sharing private information
* repeated bad-faith derailment or trolling
* spam, scams, or unrelated promotion

## Enforcement

The maintainer may edit, hide, lock, or remove content that breaks this policy and may restrict participation for repeat offenders.

## Scope

This applies to repository discussions, issues, pull requests, release threads, and other project communication channels.
